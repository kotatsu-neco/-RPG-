# グラフィック部門への正式共有・再設計依頼  
## v3.4.3フラットground診断結果を受けた背景タイル再設計について

グラフィック部門の皆様へ

お疲れさまです。  
実装部門です。

v3.4.1〜v3.4.3の実機確認結果を共有します。  
結論として、白い格子状表示・地面のガビガビ感については、**エンジン本体ではなく、groundタイル素材の視覚設計に起因する可能性が高い**と判断しました。

---

## 1. 実機確認結果

### v3.4.1

状態:

```text
起動エラーを修正後、ゲーム画面は表示。
ただし、地面・床・壁・道などに白い格子状の線が強く出た。
```

見え方:

```text
- 16×16タイルの境界が全面に出る
- 地面ではなく、升目状のパネルに見える
- タイル1枚ごとの主張が強い
```

---

### v3.4.2

対応内容:

```text
グラフィック部門から受領した 16×16完全不透明groundタイルを導入
ground / object / decoration の3層化
ground layerにはopaque=trueのground専用タイルを使用
```

結果:

```text
依然として地面の格子状表示が残った。
```

判断:

```text
alphaや透明余白だけではなく、groundタイル画像そのものの外周・質感・境界感が強すぎる可能性が高い。
```

---

### v3.4.3

対応内容:

```text
診断用として、ground layerをほぼ単色のフラットタイルへ差し替え。
object / decoration は既存素材のまま維持。
```

結果:

```text
groundの継ぎ目は消えた。
```

この結果により、実装部門としては以下を確定判断とします。

```text
地面の格子状表示の主因は、エンジン描画ではなくgroundタイル素材設計にある。
```

---

## 2. 実装部門の判断

ゲーム全体、エンジン、スマホUI、会話システム、TileMapRendererは作り直し不要です。

維持するもの:

```text
- 現在のエンジン構造
- 16×16タイル方針
- ground / object / decoration の3層構造
- TileMapRenderer
- semantic manifest
- スマホ縦画面UI
```

見直すべきもの:

```text
- groundタイルの美術設計
- object / decoration の外周白縁・余白
- 建物・家具・小物素材の切り出し精度
```

---

## 3. groundタイル再設計の依頼

groundタイルは「1枚の小さな絵」ではなく、マップ全体の**面**を作る素材として再設計をお願いします。

### 3.1 groundタイルの基本方針

以下を正式方針にしたいです。

```text
- 外周に線を入れない
- 外周1pxに明るい縁を入れない
- 隣接タイルと自然につながる
- 1枚単体で目立たせない
- 高コントラストな模様を避ける
- 質感はかなり弱くする
- 地面の情報量はdecorationへ逃がす
```

---

### 3.2 草地

望ましい方向:

```text
ほぼ単色の緑
ごく弱い粒状ノイズ
外周には模様を置かない
濃淡差を小さくする
```

避けたい方向:

```text
タイル外周に明るい線がある
中央と外周の明度差が大きい
草の模様が1枚ごとに目立つ
繰り返した時に升目に見える
```

---

### 3.3 土道

望ましい方向:

```text
落ち着いた茶色
ごく弱い土の粒
外周に線を入れない
道の縁取りは別タイルまたはdecorationに逃がす
```

避けたい方向:

```text
1タイルごとに枠がある
縁が明るい
敷き詰めるとレンガやパネル状に見える
```

---

### 3.4 石畳・石床

石系は特に格子化しやすいため、注意が必要です。

望ましい方向:

```text
石の継ぎ目を弱くする
タイル外周に継ぎ目を固定しない
細かい線を薄くする
大きな升目に見せない
```

場合によっては、石畳は以下のような運用でもよいと考えます。

```text
- groundとしては弱い灰色面
- 石の割れ目や小石はdecoration layerで散らす
```

---

### 3.5 木床

望ましい方向:

```text
板目は控えめ
外周に白縁を入れない
1タイルごとに板が完結しない
横方向・縦方向に自然につながる
```

---

## 4. decorationへの情報量移動

groundに情報量を詰め込みすぎると、画面全体が騒がしくなります。  
今後は以下のように分ける方針を推奨します。

```text
ground:
  面を作る。控えめ。

decoration:
  草、石、小物、影、苔、落ち葉、足跡などで雰囲気を足す。

object:
  家具、壁、井戸、墓石、棚、机などの存在物。
```

例:

```text
草地の細かい草 → decoration
小石 → decoration
道端の欠け → decoration
墓地の苔 → groundまたはdecorationを用途別に分ける
扉ハイライト → decoration
```

## 5. object / decoration の白縁対策について

v3.4.3では ground の継ぎ目は消えましたが、object / decoration 側にはまだ白縁や余白が見える箇所があります。

対象候補:

```text
家屋
屋根
壁
井戸
家具
棚
机
椅子
手紙の束
小物
```

これらについても、今後は以下を確認したいです。

```text
- 透明PNGの外周に白・灰色の縁がないか
- シート背景由来の色が残っていないか
- 透明部分のアンチエイリアスが白背景前提になっていないか
- 16×16内の配置基準が揃っているか
```

## 6. 今後の素材納品で希望する仕様

### 6.1 groundタイル

```text
サイズ: 16×16
alpha: 全ピクセル255
resize_rule: do_not_resize
edge_policy: no_visible_border
layer_type: ground
opaque: true
collision: false
```

### 6.2 object / decoration

```text
透明あり可
ただし外周白縁なし
背景色由来の半透明ピクセルなし
アンチエイリアスが必要な場合は暗背景・地面背景でも違和感が出ないこと
```

### 6.3 manifest

今後のmanifestには、以下の列を維持していただけると助かります。

```csv
tile_id,filename,semantic_name,category,layer_type,opaque,collision,width,height,alpha_rule,resize_rule,edge_policy,notes
```

特に追加したい列:

```text
edge_policy
```

例:

```text
no_visible_border
transparent_clean_edge
debug_only
not_for_ground
```

## 7. v3.4.4での実装側暫定方針

実装部門では、開発を止めないため、当面は以下を採用します。

```text
ground:
  v3.4.3のフラット診断タイルを暫定運用

object / decoration:
  既存素材を継続使用

目的:
  ストーリー・イベント・UI実装を先に進める
```

つまり、正式ground素材の再設計を待ちながらも、開発全体は止めません。
